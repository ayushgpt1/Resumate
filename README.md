# Resumate — AI-Powered HR Candidate Search Platform

Resumate is a full-stack web application that streamlines the recruitment process by leveraging AI to automatically parse, analyze, and rank resumes. It helps HR professionals evaluate candidates against job requirements efficiently using intelligent scoring.

## 🚀 Features

- **AI Resume Parsing & Ranking** — Upload resumes (PDF, DOCX, DOC, images, Excel) and automatically extract candidate details, skills, and experience. Each candidate is scored (0–100) based on skill alignment, experience quality, growth potential, and unique value.
- **Role Management** — Create job roles with descriptions, view all candidates associated with each role, and manage the hiring pipeline.
- **Interview Email Automation** — Send professional interview invitations via Brevo SMTP, with email bodies generated by Google Gemini AI (or a static fallback template).
- **LinkedIn Integration** — Search and retrieve candidate profiles from LinkedIn.
- **Authentication** — Secure user authentication powered by Clerk.
- **Cloud Storage** — Resume files stored in Supabase Storage, candidate data in Supabase PostgreSQL.
- **Local Fallback** — Operates with local JSON storage when Supabase is unavailable.

## 🛠️ Getting Started

### Prerequisites

- Python 3.10+
- Node.js 18+
- A **Groq API key** (for AI resume parsing) — [Get one here](https://console.groq.com)
- A **Supabase project** (PostgreSQL + Storage)
- A **Clerk application** (for authentication)
- (Optional) A **Gemini API key** (for email template generation)
- (Optional) A **Brevo API key** (for sending interview emails)

### Backend Setup

```bash
cd backend
cp env.template .env
# Fill in your .env with API keys and database credentials
pip install -r requirements.txt
python main.py
```

The API will start at `http://localhost:5000`.

### Frontend Setup

```bash
cd frontend
cp .env.example .env
# Fill in your Clerk publishable key
npm install
npm run dev
```

The frontend will start at `http://localhost:5173`.

### Environment Variables

#### Backend (`backend/.env`)

| Variable            | Required | Description                           |
| ------------------- | -------- | ------------------------------------- |
| `SUPABASE_URL`      | Yes      | Supabase project URL                  |
| `SUPABASE_KEY`      | Yes      | Supabase anon/service role key        |
| `SUPABASE_JWT_SECRET` | Yes    | Supabase JWT secret                   |
| `GROQ_API_KEY`      | Yes      | Groq API key for AI processing        |
| `CLERK_SECRET_KEY`  | Yes      | Clerk secret key (JWT verification)   |
| `GEMINI_API_KEY`    | No       | Google Gemini API key (email templating) |
| `BREVO_API_KEY`     | No       | Brevo SMTP API key (send emails)      |

#### Frontend (`frontend/.env`)

| Variable                 | Required | Description                    |
| ------------------------ | -------- | ------------------------------ |
| `VITE_CLERK_PUBLISHABLE_KEY` | Yes  | Clerk publishable key          |
| `VITE_API_URL`           | No       | Backend API URL (default: `http://localhost:5000`) |

## 📡 API Endpoints

| Method | Endpoint                              | Description                    |
| ------ | ------------------------------------- | ------------------------------ |
| POST   | `/process`                            | Upload & process resumes       |
| GET    | `/roles`                              | List all job roles             |
| GET    | `/candidates_for_role/<role_name>`    | Get candidates for a role      |
| DELETE | `/roles/<role_name>`                  | Delete a role and its candidates |
| DELETE | `/candidates/<id>`                    | Delete a candidate             |
| POST   | `/send-interview-invitation`          | Send interview email           |
| POST   | `/linkedin/candidates/search`         | Search LinkedIn candidates     |
| GET    | `/test-db`                            | Test database connectivity     |
